Omnibus GitLab documentation

Omnibus GitLab是集成了GitLab及GitLab所需的服务、依赖的一键安装包，开发者可以不用费劲的折腾各种环境、依赖、配置等，轻松的安装GitLab到流行的linux发行版系统上。

Installation 安装说明

  • 安装包下载页面 下载页面已经针对国内网络环境进行优化，镜像源由TUNA提供。
  • 安装需求
  • GitLab CI 设置Omnibus GitLab 包里面自带的GitLab CI。
  • GitLab Mattermost 设置Omnibus GitLab 包里面自带的Mattermost（Slack的代替品，团队即时通信服务，即IM）应用。
  • Docker 在Docker容器上运行GitLab.

Troubleshooting 故障排查

  • 下载包的时候提示"Hash Sum mismatch"
  • Apt报错: 'The requested URL returned error: 403'.
  • 使用浏览器无法访问GitLab页面.
  • GitLab CI显示的是GitLab登录页面(不适用于8.x及之后版本).
  • 邮件无法发送.
  • Reconfigure的时候在ruby_block[supervise_redis_sleep] action run停止运行.
  • GitLab TCP端口被占用.
  • 在启用了SELinux的系统上无法使用Git SSH访问GitLab.
  • Postgres报错 'FATAL: could not create shared memory segment: Cannot allocate memory'.
  • Reconfigure的时候关于GLIBC版本的报错.
  • Reconfigure报错"fails to create the git user".
  • 使用sysctl修改内核参数失败 .
  • 无法使用非root用户安装Omnibus GitLab.
  • gitlab-rake assets:precompile 报错提示 'Permission denied'.
  • 'Short read or OOM loading DB' 报错.
  • 'pg_dump: aborting because of server version mismatch'报错
  • 'Errno::ENOMEM: Cannot allocate memory' during backup or upgrade
  • NGINX error: 'could not build server_names_hash'
  • Reconfigure fails due to "'root' cannot chown" with NFS root_squash
